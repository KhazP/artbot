# Support

Use the right channel so maintainers and contributors can respond efficiently.

## Questions and Usage Help

Use **GitHub Discussions** for:

- setup problems
- usage questions
- architecture questions
- open-ended ideas before they become actionable issues

## Bugs and Feature Work

Use **GitHub Issues** for:

- reproducible defects
- scoped feature requests
- documentation mistakes with a clear correction path

Include enough detail to reproduce the problem or evaluate the proposal.

## Security

Do **not** use public issues or discussions for vulnerabilities.

- Preferred: private GitHub vulnerability reporting under the Security tab
- Fallback: `contact@alpyalay.org`

See [SECURITY.md](SECURITY.md) for the reporting policy.
